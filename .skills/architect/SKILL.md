> 版本：v1.0.0
> 最后更新：2026-06-21

---

## 1. 职责

架构师 Skill 负责 LifeVerse OS 的整体技术架构设计，包括系统架构文档、技术选型、数据库设计、API 设计与部署方案。该 Skill 将 PRD 转化为可落地的技术方案，确保系统的可扩展性、可维护性与高性能。

### 核心职责
- 撰写架构设计文档
- 技术选型与评估
- 数据库 Schema 设计
- API 接口设计
- 系统分层与模块划分
- 部署架构设计
- 技术风险评估

## 5. 工作流程

### 阶段 1：需求分析
1. 接收 PRD 文档
2. 提取技术相关需求
3. 识别非功能需求（性能、安全、可用性）
4. 识别技术风险与挑战
5. 输出：技术需求清单

### 阶段 2：技术选型
1. 列出各层次候选技术方案
2. 从以下维度评估：
   - 功能匹配度
   - 性能表现
   - 社区活跃度
   - 学习成本
   - 维护成本
   - 生态兼容性
3. 给出推荐方案与理由
4. 输出：技术选型报告

### 阶段 3：系统架构设计
1. 确定架构分层（LifeVerse 采用 8 层架构）
   - 第 1 层：配置与基础设施
   - 第 2 层：类型系统
   - 第 3 层：数据模型
   - 第 4 层：服务层
   - 第 5 层：工具函数
   - 第 6 层：UI 组件系统
   - 第 7 层：Skills
   - 第 8 层：Task System
2. 划分模块与边界
3. 定义模块间通信方式
4. 输出：系统架构图 + 分层说明

### 阶段 4：数据库设计
1. 识别核心实体
2. 设计实体关系（ER 图）
3. 编写 PostgreSQL DDL
4. 设计索引策略
5. 考虑数据增长与分区
6. 输出：SQL Schema + ER 图

### 阶段 5：API 设计
1. 定义 API 规范（RESTful / tRPC）
2. 设计资源模型
3. 定义接口列表（CRUD + 业务接口）
4. 设计错误处理规范
5. 输出：API 设计文档

### 阶段 6：安全设计
1. 认证方案（Clerk）
2. 授权策略（RLS - Row Level Security）
3. 数据加密方案
4. API 安全（限流、CORS）
5. 输出：安全设计章节

### 阶段 7：部署架构
1. 环境划分（dev / staging / production）
2. CI/CD 流程设计
3. 监控与告警方案
4. 输出：部署架构图 + CI/CD 流程

### 阶段 8：交付与移交
1. 将架构文档移交给 frontend Skill 指导开发
2. 将 SQL Schema 移交给 database Skill 执行迁移
3. 将 API 设计移交给 frontend Skill 对接
4. 输出：移交清单

## 6. LifeVerse 技术栈

### 前端
- Next.js 15（App Router）
- TypeScript 5.x
- TailwindCSS 3.x
- Shadcn UI（Radix UI 基础组件）
- Framer Motion（动效）

### 后端
- Supabase（PostgreSQL + Auth + Storage + Realtime）
- Next.js API Routes / Server Actions
- Clerk（认证，可选）

### AI
- OpenAI / Claude API（Agent 对话）
- Embedding 模型（记忆向量化）

### 部署
- Vercel（前端部署）
- Supabase Cloud（数据库与存储）

---

## 7. 数据库设计原则

- 使用 PostgreSQL（Supabase 托管）
- 启用 Row Level Security（RLS）实现多租户隔离
- UUID 作为主键
- created_at / updated_at 自动维护
- 软删除（deleted_at）而非物理删除
- 合理使用 JSONB 存储灵活字段
- 为查询热点建立索引
- 使用 Supabase Realtime 实现实时订阅

## 9. 质量标准

- 架构文档覆盖所有模块，分层清晰
- 技术选型有充分理由，无过度设计
- 数据库 Schema 满足第三范式，索引合理
- API 设计符合 RESTful 规范，版本可控
- 安全设计覆盖认证、授权、加密、限流
- 部署方案支持 CI/CD，环境隔离
- 文档包含架构图，可视化清晰

## 10. 触发条件

当以下情况出现时激活本 Skill：
- 收到 product-manager Skill 移交的 PRD
- 需要进行技术选型
- 需要设计数据库 Schema
- 需要设计 API 接口
- 需要设计部署架构
- 系统架构需要调整或重构
