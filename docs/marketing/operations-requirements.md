## 1. 文档说明

本文档定义 LifeVerse 运营团队所需的管理后台功能、用户运营策略与社区运营计划，是产品研发团队实现运营后台的依据，也是运营团队开展日常工作的规范。

### 1.1 运营目标

| 目标 | 衡量方式 |
| --- | --- |
| 让用户快速体验核心价值 | 首次议会完成率 ≥ 60% |
| 让用户持续回来 | 7 日留存 ≥ 35%，30 日留存 ≥ 30% |
| 让用户愿意付费 | 付费转化率 ≥ 3%（MVP） |
| 让用户愿意传播 | 分享率 ≥ 8% |
| 让用户感到被关注 | 反馈响应时长 ≤ 24h |

### 1.2 运营团队角色

| 角色 | 职责 |
| --- | --- |
| 运营负责人 | 运营策略制定、数据监控、跨部门协调 |
| 用户运营 | 新手引导、活跃激励、沉默召回、付费转化 |
| 内容运营 | 内容矩阵执行、UGC 管理、素材库维护 |
| 社区运营 | 社群搭建、话题运营、活动策划、用户共创 |
| 数据运营 | 数据看板维护、漏斗分析、增长实验 |

## 2. 管理后台运营需求

### 2.1 公告管理

#### 2.1.1 功能概述

运营人员可在后台发布站内公告，支持图文、定时发布、置顶，用于产品更新通知、活动公告、重要提示等。

#### 2.1.2 功能需求

| 编号 | 需求 | 优先级 | 说明 |
| --- | --- | --- | --- |
| AN-001 | 公告创建 | P0 | 支持标题、正文（富文本）、封面图、跳转链接 |
| AN-002 | 公告类型 | P0 | 支持分类：产品更新、活动公告、系统通知、紧急通知 |
| AN-003 | 定时发布 | P0 | 支持设定发布时间，到点自动上线 |
| AN-004 | 置顶管理 | P0 | 支持置顶/取消置顶，最多同时置顶 3 条 |
| AN-005 | 定向下发 | P1 | 支持按用户标签（新用户/付费用户/沉默用户）定向推送 |
| AN-006 | 公告编辑 | P0 | 支持已发布公告的编辑与撤回 |
| AN-007 | 公告列表 | P0 | 列表展示所有公告，支持按状态/类型/时间筛选 |
| AN-008 | 效果统计 | P1 | 统计每条公告的曝光数、点击数、关闭率 |
| AN-009 | 多端适配 | P0 | 公告在 Web 端与 PWA 端均正常展示 |
| AN-010 | 用户关闭记忆 | P1 | 记录用户已关闭的公告，不再重复弹出 |

#### 2.1.3 公告展示规范

- 新公告首次登录时以弹窗形式展示（可关闭）。
- 公告入口常驻首页右上角铃铛图标，红点提示未读。
- 紧急通知（如服务故障）以顶部横幅形式强展示，不可关闭直至解除。

### 2.5 内容审核

#### 2.5.1 功能概述

对用户生成内容（UGC）进行审核，包括分享的议会报告、社区帖子、用户共创智者建议等。

#### 2.5.2 审核范围

| 内容类型 | 审核方式 | 说明 |
| --- | --- | --- |
| 分享议会报告卡片 | AI 初审 + 人工抽审 | 检查是否含敏感信息、是否脱敏 |
| 社区帖子 | AI 初审 + 人工复审 | 检查违规内容、广告、人身攻击 |
| 用户共创智者建议 | 人工审核 | 检查智者画像合规性 |
| 用户反馈 | AI 初审 | 检测自伤倾向等紧急情况 |
| AI 亲人命名/描述 | AI 初审 | 检查是否涉及在世真人 |

#### 2.5.3 功能需求

| 编号 | 需求 | 优先级 | 说明 |
| --- | --- | --- | --- |
| AU-001 | 审核队列 | P0 | 待审核内容按时间倒序排列 |
| AU-002 | 审核操作 | P0 | 支持通过、拒绝、打回修改、标记违规 |
| AU-003 | 审核理由 | P0 | 拒绝时需填写理由，通知用户 |
| AU-004 | 敏感词库 | P0 | 维护敏感词库，AI 自动标记 |
| AU-005 | 心理危机检测 | P0 | 检测自伤/自杀倾向，触发危机干预流程 |
| AU-006 | 批量审核 | P1 | 支持批量通过/拒绝 |
| AU-007 | 审核记录 | P0 | 记录每次审核的操作人与操作结果 |
| AU-008 | 申诉处理 | P1 | 用户可对审核结果申诉，运营复审 |
| AU-009 | 审核统计 | P1 | 统计审核量、通过率、违规率 |

#### 2.5.4 审核流程

```
用户发布内容
    ↓
AI 初审（敏感词 + 心理危机检测）
    ↓
┌─ 通过 → 内容上线
├─ 疑似违规 → 人工复审 → 通过/拒绝
├─ 检测到心理危机 → 触发危机干预流程（详见 2.5.5）
└─ 明确违规 → 拒绝 + 通知用户
```

#### 2.5.5 心理危机干预流程

当 AI 检测到用户内容含自伤/自杀倾向时：

1. 立即标记为"紧急"，通知运营值班人员（5 分钟内响应）。
2. 系统自动向用户推送心理援助热线（全国 24 小时心理危机干预热线：400-161-9995）。
3. 运营人员人工评估，必要时联系专业心理机构转介。
4. 事件记录归档，纳入心理安全报告。

## 3. 用户运营策略

### 3.1 新用户引导流程（Onboarding）

#### 3.1.1 引导目标

让新用户在 5 分钟内完成自我画像，在 15 分钟内完成第一次议会，体验核心价值。

#### 3.1.2 引导流程

```
注册成功
    ↓
Step 1: 欢迎页（10 秒）
"欢迎来到你的宇宙。接下来 5 分钟，我们先认识一下你。"
    ↓
Step 2: 基本背景（1 分钟）
- 昵称、年龄段、职业、城市
- 当前情绪状态（滑块）
    ↓
Step 3: 价值雷达（2 分钟）
- 五维价值雷达可视化，用户拖动滑块调整
- 实时反馈："你是一个重视成长与自由的人"
    ↓
Step 4: 当前困境（1 分钟）
- "你现在面临什么人生决策？"（可选填）
- 预设选项：职业、关系、城市、学业、其他
    ↓
Step 5: 自我画像卡片生成（10 秒）
- 生成可分享的自我画像卡片
- "这是你的宇宙的第一颗星。"
    ↓
Step 6: 首次议会引导（30 秒）
- "要不要让 7 位智者，为你刚才提到的困境召开一场议会？"
- 一键进入智慧议会
    ↓
Step 7: 首次议会体验（10~15 分钟）
- 引导输入/选择议题
- 7 智者议会完整流程
- 生成命运报告
    ↓
Step 8: 首次议会后引导（30 秒）
- "这是你的第一份命运报告，已写入你的生命时间线。"
- 引导查看 History + 引导分享 + 引导召开未来议会
```

#### 3.1.3 引导优化指标

| 指标 | 目标 | 优化方向 |
| --- | --- | --- |
| 注册→自我画像完成率 | ≥ 90% | 简化问题、增加进度条 |
| 自我画像→首次议会完成率 | ≥ 70% | 强化引导、降低议题输入门槛 |
| 首次议会满意度 | ≥ 4.0/5 | 优化智者发言质量与报告生成 |
| 引导总时长 | ≤ 20 分钟 | 精简流程、提供跳过选项 |

### 3.2 活跃用户激励

#### 3.2.1 每日议会提醒

- 用户可开启"每日议会提醒"，每日固定时间推送。
- 推送内容含"今日议题建议"（基于用户画像与历史议会生成）。
- 示例："今日议题：你最近一直在回避的那件事，要不要让智者们帮你看看？"

#### 3.2.2 成就系统

| 成就 | 获取条件 | 奖励 |
| --- | --- | --- |
| 初次议会 | 完成第一次议会 | 自我画像卡片升级 |
| 七智者之友 | 召开 7 次智慧议会 | 专属头像框 |
| 时间旅人 | 召开 3 次未来议会 | 价值雷达漂移轨迹解锁 |
| 议会大师 | 累计召开 30 次议会 | 1 个月 Pro 体验 |
| 价值觉察者 | 价值雷达漂移 5 次 | 觉察者徽章 |
| 分享者 | 分享 3 次议会报告 | 分享者徽章 + 1 次额外议会 |
| 宇宙建造者 | 上传 100 条记忆 | 人生地图专属皮肤 |
| 重逢者 | 完成 1 次重逢对话 | 重逢者徽章 |
| 年度回望者 | 收到年度生命报告 | 年度报告专属封面 |

#### 3.2.3 议会连续打卡

- 连续 7 天召开议会：获"一周觉察者"徽章 + 1 次额外议会。
- 连续 30 天召开议会：获"月度觉察者"徽章 + 1 个月 Pro 体验。
- 打卡不强制每日，每周至少 1 次议会即算"活跃"。

#### 3.2.4 价值雷达漂移追踪

- 每次议会后更新价值雷达，用户可查看漂移轨迹。
- 漂移幅度大时主动提示："你的价值观正在发生变化，点击查看。"
- 月度生成"价值漂移报告"，让用户看见自己的成长。

### 3.3 沉默用户召回

#### 3.3.1 召回分层策略

| 沉默等级 | 定义 | 召回渠道 | 召回内容 | 目标召回率 |
| --- | --- | --- | --- | --- |
| 轻度沉默 | 3 天未登录 | 站内信 + 浏览器推送 | "你的智者们在等你" + 议会入口 | 15% |
| 中度沉默 | 7 天未登录 | 邮件 | "80 岁的你有一封信" + 新议题建议 | 10% |
| 重度沉默 | 30 天未登录 | 邮件 + 微信服务号 | "你的宇宙还在这里" + 新功能介绍 + 优惠 | 5% |
| 流失用户 | 90 天未登录 | 邮件 | "我们想你了" + Pro 体验月 | 3% |

#### 3.3.2 召回内容设计原则

- **不制造焦虑**：召回文案温暖而非催促，避免"你怎么还不来"。
- **提供新价值**：每次召回附带新议题/新功能/新智者，给回来的理由。
- **尊重选择**：连续 3 次召回未响应，降低推送频次至每月 1 次。
- **一键退订**：每封召回邮件含退订按钮。

#### 3.3.3 召回效果追踪

- 每个召回批次追踪：送达率、打开率、点击率、回访率、回访后留存率。
- 月度复盘召回效果，优化文案与时机。

## 4. 社区运营计划

### 4.1 社区定位

LifeVerse 社区是一个"自我觉察者的聚集地"，用户在这里分享议会实录、讨论人生决策、共建智者生态。

### 4.2 社区平台搭建

#### 4.2.1 Discord 社区（海外 + 技术用户）

| 频道 | 用途 |
| --- | --- |
| #welcome | 新人欢迎与引导 |
| #announcements | 产品公告 |
| #wisdom-council | 智慧议会讨论 |
| #future-council | 未来议会讨论 |
| #my-council | 分享议会实录 |
| #life-decisions | 人生决策讨论 |
| #ai-philosophy | AI 与自我觉察哲学讨论 |
| #feature-requests | 功能建议 |
| #bug-reports | Bug 反馈 |
| #off-topic | 闲聊 |

#### 4.2.2 微信社群（国内核心用户）

| 群类型 | 人数 | 用途 |
| --- | --- | --- |
| LifeVerse 体验群 | 200 人/群，多群 | 日常交流、互助、反馈 |
| LifeVerse Pro 用户群 | 100 人 | 付费用户专属、深度讨论 |
| LifeVerse 种子用户群 | 50 人 | 早期用户、产品共创 |
| LifeVerse 创作者群 | 30 人 | 内容创作者、UGC 激励 |

#### 4.2.3 社区管理规范

- 每个群配备 1 名管理员 + 2 名志愿者。
- 制定社区公约：尊重、真实、不广告、不人身攻击。
- 违规处理：警告 → 禁言 → 移出。
- 心理安全：发现自伤倾向立即私聊干预 + 触发危机流程。

### 4.3 每周话题讨论

#### 4.3.1 话题机制

- 每周一发布本周话题，在社区与全渠道同步。
- 话题来自"10 个预设议会话题"轮换 + 热点结合。
- 鼓励用户召开议会后分享实录，参与讨论。

#### 4.3.2 首月话题排期

| 周次 | 话题 | 讨论引导 |
| --- | --- | --- |
| 第 1 周 | 如果可以问 80 岁的自己一个问题，你会问什么？ | 引导未来议会体验 |
| 第 2 周 | 你做过最不后悔的决定是什么？ | 引导后悔分析讨论 |
| 第 3 周 | 7 位智者中，谁最像你？ | 引导价值雷达讨论 |
| 第 4 周 | 如果人生可以重来一次，你会改变哪个决定？ | 引导扰动时间线讨论 |

### 4.4 月度最佳议会评选

#### 4.4.1 评选机制

- 每月最后一周，运营从当月用户分享的议会实录中提名 10 个。
- 社区用户投票选出"月度最佳议会"Top 3。
- 评选维度：议题共鸣度、智者辩论质量、决策启发性。

#### 4.4.2 奖励

| 名次 | 奖励 |
| --- | --- |
| 第 1 名 | 3 个月 Pro 体验 + 官方公众号专访 + 限定徽章 |
| 第 2 名 | 2 个月 Pro 体验 + 限定徽章 |
| 第 3 名 | 1 个月 Pro 体验 + 限定徽章 |
| 提名奖 | 1 个月 Pro 体验 |

#### 4.4.3 内容再利用

- 获奖议会实录经脱敏后作为官方内容素材发布。
- 获奖用户可成为"LifeVerse 体验官"，参与新产品内测。

### 4.5 用户共创：智者建议

#### 4.5.1 共创机制

允许用户提交新的智者建议，丰富智者生态。

#### 4.5.2 提交流程

```
用户提交智者建议
    ↓
填写智者画像表单：
- 智者姓名（历史人物/虚构角色）
- 时代与领域
- 核心哲学（100 字以内）
- 价值雷达坐标
- 典型发言风格
- 擅长议题
    ↓
运营审核（合规性、独特性、与现有智者的差异化）
    ↓
┌─ 通过 → 进入"社区智者"候选池
├─ 需修改 → 打回用户修改
└─ 拒绝 → 通知理由
    ↓
社区投票（候选智者获 100+ 票）
    ↓
官方采纳 → 上线为"社区智者"
    ↓
提交者获"共创者"徽章 + 3 个月 Pro 体验
```

#### 4.5.3 共创规范

- 智者必须是已故历史人物或明确虚构角色，不复活在世者。
- 智者画像需尊重历史，不歪曲核心哲学。
- 智者需与现有 7 位智者有差异化，不重复。
- 每季度最多上线 2 位社区智者，保证质量。
