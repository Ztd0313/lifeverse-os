## 1. 内容运营总纲

### 1.1 内容使命

用内容建立"AI 生命操作系统"的品类认知，让目标用户在面临人生决策时，第一个想到 LifeVerse。

### 1.2 内容目标

| 目标 | 衡量指标 |
| --- | --- |
| 建立品类认知 | 品牌词搜索量、内容曝光量 |
| 获取种子用户 | 内容→注册转化率 |
| 验证产品价值 | 用户故事产出量、UGC 议会实录 |
| 沉淀长期资产 | SEO 流量、内容库规模 |

### 1.3 内容原则

1. **真实优先**：用户议会实录 > 编造故事。真实决策困境比虚构案例更有共鸣。
2. **价值优先**：每条内容必须独立提供价值（一个洞察/一个方法/一次共鸣），而非纯广告。
3. **场景优先**：内容从"决策场景"切入，而非从"功能"切入。用户不关心功能，关心自己的困境。
4. **多元优先**：展现 7 智者的多元视角，让内容本身就是"议会"的缩影。

## 4. 内容素材库

### 4.1 预设议会话题（10 个）

以下 10 个话题作为内容生产的"弹药库"，每个话题可衍生多条内容（议会实录、用户故事、方法论、短视频）。

#### 话题 1 · 辞职创业

- **议题**："我是否应该辞职创业？"
- **典型用户**：28 岁互联网产品经理
- **智者冲突点**：马斯克（支持）vs 巴菲特（反对）vs 庄子（放下）
- **冲突值预估**：0.71（风暴级）
- **内容角度**：能力圈、第一性原理、自由与稳定、后悔分析
- **适用渠道**：小红书、知乎、B 站

#### 话题 2 · 是否结婚

- **议题**："我是否应该和现在的伴侣结婚？"

#### 话题 3 · 是否生育

### 4.2 用户故事模板（5 个）

#### 模板 1 · 决策转变型

```
标题：《[人物特征]的[TA]，让[智者/时间自己]做了一个[年龄]的决定》

结构：
1. 困境引入：用 2~3 句话描述用户面临的决策困境，引发共鸣。
2. 传统方式失效：描述用户尝试过的传统方法（问朋友、列优缺点）为何无效。
3. LifeVerse 介入：用户如何发现 LifeVerse，召开了什么议会。
4. 议会精华：摘录 2~3 位智者/时间自己的关键发言（脱敏）。
5. 转变时刻：用户在议会中获得的"啊哈"时刻。
6. 最终决定：用户做了什么决定（或决定暂不做决定）。
7. 回望寄语：3 个月后用户的回望感受。

字数：1,500~2,500 字
```

#### 模板 2 · 自我觉察型

```
标题：《[TA]在 LifeVerse 发现：原来自己是[洞察]》

结构：
1. 表面困惑：用户以为自己的问题是 A，其实是 B。
2. 价值雷达：通过价值雷达看见自己的真实价值排序。
3. 议会揭示：智者/时间自己如何点破盲区。
4. 觉察转变：用户从"纠结决定"到"理解自己"的转变。
5. 新的行动：基于觉察的新行动。

字数：1,200~2,000 字
```

#### 模板 3 · 关系疗愈型

```
标题：《[TA]与[关系]的重逢：一场迟到的[告别/和解/感恩]》

结构：
1. 未完成的关系：描述一段未完成的关系（亲人离世/关系破裂）。
2. 遗憾的重量：这个遗憾如何影响用户的生活。
3. Reunion 介入：用户如何生成 AI 亲人，进行了什么对话。
4. 重逢时刻：对话中最触动的一句话。
5. 释怀与前行：用户如何带着这份释怀继续生活。

字数：1,500~2,500 字
注意：此类故事需用户明确授权，严格脱敏，强调 AI 身份声明。
```

#### 模板 4 · 未来回望型

```
标题：《[TA]问 80 岁的自己：[问题]》

结构：
1. 当下焦虑：描述用户当下的短期焦虑。
2. 未来议会：用户召开未来议会，4 个时间自己发言。
3. 80 岁的回望：80 岁自己的关键发言（最打动人的部分）。
4. 视角转换：用户从"当下焦虑"到"长期回望"的视角转换。
5. 后悔分析：用户更愿意承担哪种后悔。
6. 新的从容：用户带着长期视角重新面对当下。

字数：1,200~2,000 字
```

#### 模板 5 · 长期成长型

```
标题：《使用 LifeVerse [N]个月后，[TA]的价值雷达变了》

结构：
1. 初始画像：用户 N 个月前的价值雷达与状态。
2. 议会历程：期间召开的关键议会（3~5 次）。
3. 雷达漂移：价值雷达如何变化，意味着什么。
4. 成长洞察：用户对自己成长的最大感悟。
5. 年度寄语：80 岁自己想对现在的 TA 说的话。

字数：1,500~2,500 字
```

## 5. 内容生产流程

### 5.1 内容生产 SOP

```
选题 → 大纲 → 初稿 → 审核 → 设计 → 发布 → 监测 → 复盘
 1天    1天    2天    1天    1天    当天   持续    每周
```

### 5.2 选题来源

| 来源 | 占比 | 说明 |
| --- | --- | --- |
| 议会话题库 | 40% | 从 10 个预设话题衍生 |
| 用户反馈 | 25% | 从用户议会实录与反馈中挖掘 |
| 热点借势 | 15% | 社会热点/节日/行业事件结合 |
| 数据驱动 | 10% | 从高表现内容中提炼模式复制 |
| 创始人视角 | 10% | 产品思考与 AI 哲学 |

### 5.3 内容审核标准

所有内容发布前需通过以下审核：

| 审核项 | 标准 | 负责人 |
| --- | --- | --- |
| 事实准确 | 智者观点、产品功能描述准确无误 | 内容策略师 |
| 隐私脱敏 | 用户故事已脱敏，已获授权 | 内容策略师 |
| 伦理合规 | 无心理风险暗示，敏感话题有专业提示 | 市场总监 |
| 品牌一致 | 符合品牌调性（镜子而非裁判） | 市场总监 |
| 渠道适配 | 格式、长度、风格适配目标渠道 | 内容策略师 |

### 5.4 内容效果评估

| 指标 | 权重 | 说明 |
| --- | --- | --- |
| 曝光量 | 20% | 内容触达人数 |
| 互动率 | 20% | 点赞/评论/收藏/转发 |
| 注册转化 | 30% | 内容→注册的转化 |
| 分享率 | 15% | 内容被二次传播 |
| SEO 价值 | 15% | 长尾搜索流量贡献 |

每周复盘内容效果，淘汰末位 20%，复制头部 20% 的模式。

## 6. 持续运营机制

### 6.1 内容日历管理

- 使用 Notion 管理内容日历，所有内容提前 2 周排期。
- 每周一内容例会，复盘上周 + 确认本周 + 规划下周。
- 每月初输出月度内容计划，与产品发布节奏对齐。

### 6.2 内容与产品联动

| 产品节点 | 内容联动 |
| --- | --- |
| 新模块上线 | 提前 2 周预热 + 上线周深挖 + 上线后用户故事 |
| 新智者上线 | 智者画像专题 + 议会实录 |
| 功能优化 | 产品更新公告 + 使用技巧 |
| 年度报告 | 年度内容大片 + 用户年度故事 |

### 6.3 UGC 激励

- 鼓励用户分享"脱敏议会报告卡片"，分享即获 1 次额外议会。
- 每月评选"最佳议会实录"，获奖用户获 1 个月 Pro 体验。
- 用户故事被采用，获 3 个月 Pro 体验 + 产品致谢。
